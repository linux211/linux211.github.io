1 常见的linux开机设置文件。

/etc/fstab与/etc/initab、/etc/rc.local三个配置文件的作用？

参考答案

  • /etc/fstab：实现开机自动挂载设备的配置文件
  • /etc/initab：定义开机进入默认级别的配置文件
  • /etc/rc.local：定义开机自定义任务的配置文件

2 FTP协议简介。

FTP服务器在传输层使用的协议、默认端口、FTP主目录绝对路径？

参考答案

  • FTP服务器在传输层使用的协议是tcp
  • 默认的端口号为21
  • FTP主目录为/var/ftp

3 Linux中的udev规则。

在linux中网卡udev命名的配置文件存放在哪里？在VMware虚拟机里，网卡驱动名称是什么？

参考答案

  • 网卡udev命名的配置文件：/etc/udev/rules.d/70-persistent-net.rules
  • 在VMware虚拟机里，网卡驱动名为：e1000

5 Linux常见故障及修复。

一台Linux服务器启动时停滞在grub> 处，无法进入系统，请分析导致此故障的原因是什么，如何修复？

参考答案

出现这个现象时，可能是GRUB启动菜单的主配置文件遭到损坏。解决办法如下：

1）在grub>界面时，手动引导操作：

grub> root (hd0,0)
grub> kernel /vmlinuz-2.6.32-431.el6.x86_64 ro root=/dev/sda2
grub> initrd /initramfs-2.6.32-431.el6.x86_64.img
grub> boot

2）成功进入系统后，重建grub.cponf配置文件（可参考同系统的其他服务器）

   [root@svr5 ~]# grep -v "^#" /boot/grub/grub.conf 
   default=0
   timeout=5
   title Red Hat Enterprise Linux Server (2.6.18-348.el5)
       root (hd0,0)
       kernel /vmlinuz-2.6.32-431.el6.x86_64 ro root=/dev/sda2
       initrd /initramfs-2.6.32-431.el6.x86_64.img

必要时，还可以重装GRUB引导程序：

 [root@svr5 ~]# grub-install /dev/sda
